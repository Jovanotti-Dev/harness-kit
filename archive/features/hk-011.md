# hk-011 · Handle repos that already have a harness  *(archived)*

## Found by

Running `create` against a real iOS repo that already carried a harness-creator harness. Four
bugs surfaced in one run; none could have been found by dogfooding on harness-kit itself,
because that repo started empty.

## Done when (all met)

`create` cannot silently produce a harness the agent will never read, the audit reports that
state honestly, and there is a documented migration path that loses nothing.

## The bug that mattered

The old `CLAUDE.md` was a 72-line instruction file. `create` skipped it as "exists" and
reported success — leaving two competing harnesses. Claude Code auto-loads `CLAUDE.md`, so the
agent would follow the OLD workflow and never read anything just generated. The generator
declared victory over something functionally dead, and the audit called it 97/100.

## Decisions

- **Never delete on migration.** An old `progress.md` can hold hundreds of lines of decisions
  and root causes — the most valuable content in the repo. Old files move to `archive/legacy/`.
- **The script detects; the agent migrates.** Classifying prose as rule vs lesson vs history
  needs judgement no script has.
- **A critical failure caps the score at 49.** Averaging one wiring failure against 30 passing
  checks produced "Excellent" for an unreachable harness. When the agent never loads the file,
  the other checks are moot.
