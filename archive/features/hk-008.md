## Bug found + fixed · unresolved placeholders crashed generation

Flutter and Android both threw on first run. Probes returned `null` because the toolchains
are not installed on this machine, so `{{flutterVersion}}` and friends survived rendering and
`assertNoPlaceholders` aborted. Scaffolding a Flutter harness on a machine without Flutter is
a normal thing to want, so create.mjs now defaults any failed probe to `"not detected"`.

Second pass: `- not detected` as a bare bullet was useless, so platform lines in all four
profiles are labelled (`- Flutter toolchain: {{flutterVersion}}.`) and tell the reader to fill
the value in.
