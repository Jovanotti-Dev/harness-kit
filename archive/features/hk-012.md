## Bug found + fixed · markdown separator `|:-:|` parsed as data

The new parser tests immediately found a real defect. The separator-row regex required two or
more dashes (`^:?-{2,}:?$`), so a valid single-dash aligned separator — `|:-:|` — was not
recognised and the separator row was parsed as a **data row**.

harness-kit's own `FEATURES.md.template` emits exactly that form for evidence tables, so every
generated evidence table has been parsed with a phantom first row since the template was
written. Impact today is nil because no audit check reads `evidenceRows` yet — but it would
have produced confidently wrong results the moment one did.

Fixed to `^:?-+:?$`. This is precisely the class of silent defect the parser tests existed to
catch, found within minutes of writing them.
